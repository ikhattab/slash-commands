# Add Documentation

Add documentation to the selected code:

1. Function/Method Docs: Purpose, parameters, return values, exceptions
2. Class Docs: Class purpose and usage examples
3. Inline Comments: Only for complex or non-obvious logic
4. Type Annotations: If the language supports them

Style Guidelines:
- Match the documentation style already used in the codebase
- Be concise but complete
- Include @example where helpful
- Document edge cases and gotchas

Do NOT over-document obvious code. Focus on the "why" not the "what".
